Hugo blog 설치 및 github deploy

Tue, Mar 9, 2021 3-minute read

Hugo

정적 Blog 생성기. 비슷한 포지션에 jekyll 이 있다. 자료는 jekyll 이 많으나 ruby 로 구현 되어 있어서 go 언어로 구현되어있는 Hugo 를 선택하였다.

이번 포스팅의 내용은 아래와 같다.

  • Step1. Download
  • Step2. Blog 만들기
  • Step3. 테마 설정
  • Step4. Blog 설정
  • Step5. 새로운 Post 쓰기
  • Step6. Local 에서 확인
  • Step7. Github 에 올려 확인

Step1. Download

MAC OS 기준brew 를 사용하여 설치 한다.

brew install hugo

Step2. Blog 만들기

hugo new site BLOG

현재 위치에 BLOG 라는 디렉토리가 만들어 지고 정적으로 Blog 를 생성하기 위한 기본 빼대가 만들어 진다.

BLOG
├── archetypes
├── content    << 생성한 글들이 위치
├── data
├── layouts
├── static    << 포스팅에서 사용할 이미지 위치
└── themes    << 적용할 테마 들 위치
config.toml   << config 파일

빼대가 만들어 졌으면 git 초기화실시.

git init

BLOG 디렉토리가 git 초기화 되어 있어야 theme 나 정적으로 생성한 public 이라는 디렉토리를 관리 할수 있다.

BLOG 의 빼대만 있는 상태이고 정적으로 BLOG 를 성성 하지 않아서 public 디렉토리가 생성 전이다.

Step3. 테마 설정

https://themes.gohugo.io/

사이트에 들어가 테마를 고른다. ( 위쪽에 위치한 테마일수록 최신 이다. )

git submodule 로 테마를 가져온다.

git submodule add https://github.com/lxndrblz/anatole.git themes/anatole

Step4. Blog 설정

대부분의 테마는 exampleSite 를 제공한다. example config 파일도 같이 제공하니 해당 샘플을 blog base 위치에 가져와 자신의 정보의 맞게 수정하여 사용한다.

cp -rfp themes/anatole/exampleSite/config.toml .

config.toml 파일에서 baseurl 부분 삭제한다. 보통 example.com 으로 되어 있는데 그대로 Blog 를 생성하면 생성된 blog 가 깨지니 다른 설정은 나중에 바꾸더라고 baesurl 은 먼저 변경하자.

baseURL = ""

github 에 개시후 본인의 url 로 변경해도 된다.

해당 테마에서 전용으로 사용하는 파일들이 있을수 있으니 static 디렉토리도 복사한다.

cp -rfp themes/anatole/exampleSite/static .

Step5. 새로운 Post 쓰기

hugo new post/hugo.md

~/blog/content/post/hugo.md created

hugo new 명령으로 파일이 하나 생성 된다. 해당 파일은 archetypes 디렉토리 안에 형식으로 생성된다. ( blog 밑에 archetypes 또는 테마 디렉토리 안에 )

파일 위치는 post 밑으로 하고 파일 확장자는 꼭 마크다운 ( .md / .markdown ) 으로 생성 해야 한다.

-D 옵션은 Drafts 파일 까지 다 보여줌.

Drafts 파일이란 작업중인 글을 말한다.

테마설정에 따라 없을수도 있다.

Step7. Github 에 올려 확인

hugo 사이트에 보면 github 저장소를 2개를 만들어서 관리 하라고 권장한다.

하지만 실제적으로 deploy 는 정적으로 생성된 public 위치만 github 에 올려도 동작 한다.

Github 에 저장소 만들기

github 에 [githubID].github.io.git 의 이름으로 저장소 하나를 만든다. 만들때 readme.md 파일 생성 옵션을 클릭하여 생성 한다. ( 빈 저장소를 git submodule 로 등록하면 github 문제가 될수 있다. )

submodule 등록

public 디렉토리를 git submodule 로 등록 전에 public 디렉토리가 있으면 삭제 한다.

git submodule add https://github.com/[githubID]/[githubID].github.io.git public

이제 public 디렉토리가 생성 되었다.

정적 Blog 생성

hugo 명령으로 BLOG 의 빼대와 생성한 컨텐츠를 가지고 public 디렉토리에 html 이 정적으로 생성 된다.

github 에 올리기

public 디렉토리에 가서

git add .

git commit -m “First Posting”

git push origin master

github 에 올려거 인터넷으로 확인 하기

push 가 완료 되면 브라우져에서 url 로 확인 가능하다.
